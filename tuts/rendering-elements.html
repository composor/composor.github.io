Rendering Elements

Elements are the smallest building blocks for your app.

As was explained in the introduction to JSX, a JSX tag describes a piece of markup to be converted into DOM nodes.

            
import {h, render} from 'composi'

const element = <h1>A Title</h1>

Elements are merely a piece of markup. Notice there is no function and no functionality. Components on the other hand consist of functions or classes and can container many possible features. Elements are not components, but components are made up of elements.

Rendering an Element into the DOM

Virtual Dom

When you inject an element into the DOM with the render function, it also creates a virtual dom of the element. That means that if your element had dynamic content that could change, the render function would diff and patch the element if its data changed.

See the Pen Composi Tuts - JSX-6 by Robert Biggs (@rbiggs) on CodePen.

Composi Only Updates What is Necessary

Because of its virtual dom, Composi will only update the parts of the DOM that needs to. In many cases this will only be text nodes, so there is minimal layout thrashing. In the above example, only the text node for the time will be updated, even though we are passing in the complete element at each tick.
